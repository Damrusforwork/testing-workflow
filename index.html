⚙ Generic — 4-Skill Seamless Workflow

แต่ละ skill รับ parameter (file path) จาก user → ต่อกันแบบไร้รอยต่อ สามารถใช้กับ project ใดๆ ได้

📋 Pipeline ที่ถูกต้อง

01-create-checklist ➜ 02-test-checklist ➜ 03-fix-from-report ➜ 04-regression-test
target file➜ checklist➜ test-report➜ fix-report➜ regression-report

📌 วิธีเรียกใช้ (chain ที่ถูกต้อง)

  1. สร้าง checklist: "ใช้ skill 01-create-checklist กับ file menu source-dir=src/ output-dir=plan/test-flow/" → plan/test-flow/{date}-menu-checklist.md
  2. ทดสอบ: "ใช้ skill 02-test-checklist กับ file plan/test-flow/{date}-menu-checklist.md output-dir=reports/" → reports/test-report-menu.md
  3. แก้ไข: "ใช้ skill 03-fix-from-report กับ file reports/test-report-menu.md build-cmd='npm run build' project-dir=./ output-dir=reports/" → reports/fix-report-menu.md
  4. ตรวจสอบซ้ำ: "ใช้ skill 04-regression-test กับ file plan/test-flow/{date}-menu-checklist.md, reports/test-report-menu.md, reports/fix-report-menu.md output-dir=reports/" → reports/regression-report-menu.md

📝 01-create-checklist

input: target, source-dir (opt), output-dir (opt), patterns (opt)
output: {{output-dir}}/{date}-{target}-checklist.md
ใช้ skill 01-create-checklist กับ file menu source-dir=src/

🧪 02-test-checklist

input: checklist, output-dir (opt)
output: {{output-dir}}/test-report-{target}.md
ใช้ skill 02-test-checklist กับ file checklist-menu.md

🛠️ 03-fix-from-report

input: report, build-cmd (opt), project-dir (opt), output-dir (opt)
output: {{output-dir}}/fix-report-{target}.md
ใช้ skill 03-fix-from-report กับ file test-report-menu.md

✅ 04-regression-test

input: checklist, original-report, fix-report, output-dir (opt)
output: {{output-dir}}/regression-report-{target}.md
ใช้ skill 04-regression-test กับ file checklist-menu.md, test-report-menu.md, fix-report-menu.md
